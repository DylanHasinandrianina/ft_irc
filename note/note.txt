IRC : Internet Relay Chat

RESUME : un serveur qui recois des message, comprend des commandes, et redistribue les infos a plusieurs client en temps reel.
- L’objectif est de comprendre le modèle client-serveur, la gestion des sockets, la communication en réseau, 
    et surtout, la gestion de connexions multiples 

notre serveur IRC doit etre capable de :
    - accepter plusieurs client simultanement
    - authentifier les clients
    - gerer des channels
    - transmettre des messages
    - gerer des permission et operateurs


on peut diviser le travaille en 3 parties:
1- le reseau / poll:
    - socket(), bind(), listen(), accept(), poll(), non-blocking 
2- parsing / commande:
    - recv(), buffer, parsing, PASS, NICK, USER, JOIN, PRIVMSG
3- channels/ permission
    - channels classe, operators, KICK, INVITE, MODE, TOPIC

I - RESEAU / POLL: responsable de la creation du reseau/communication entre le client et le serveur 
    - socket() : cree une socket TCP, une porte(prise reseau) de communication entre deux programme (client --- socket --- serveur)
    - bind() : associe le socket a une adresse et un port, pour qu'apres le client saura ou se connecter
    - listen() : met le socket en mode attente de connexions
    - accept() : accepte un client, retourne une nouvelle socket dedie au client accepte(NB: chaque client possede sa propre socket)
    - connect() : cote client pour se connecter au reseau -> PAS UTILE ICI
    - send() : envoie des donnees (serveur -> client ou client -> serveur)
    - recv() : recois des donnees venant du reseau
    - close() : ferme le socket (client deconnecte)
    - poll() : TRES IMPORTANT, permet de surveiller plusieurs sockets en meme temps sans bloquer le serveur
    - fcntl() : permet mode non bloquant des sockets

II- PARSING / COMMANDES: lit les donnees recu du client, reconstruction des messages, comprehension des commandes IRC, validation
    - recv() : recevoir les entrees des client et les valide apres (NB: peux arriver en morceaux)
    - buffer : stocke temporairement les donnees recues pour reconstruire les commandes completes
    - parsing : decoupe les commandes IRC 
        ex : PRIVMSG #general :hello
        doit devenir : commande : PRIVMSG, channel : #general, message: hello
    COMMANDES PRINCIPALES :
    - PASS : le mot de passe qui doit etre pareil que l'argument
        ex : le client envoie PASS password. le serveur verifie si le mdp est correcte
    - NICK : le pseudo du client
    - USER : le username du client
    - JOIN : permet un client de rejoindre un channel
    - PRIVMSG : envoie un message prive ou dans un channel

III- CHANNELS / PERMISSION : gere le salon, envoie les message a plusieurs clients, gere les roles/droit d'utilisateurs(operators/client normal)
    - channels classe: une classe pour le salon
    - gestion user: operators(admin), client normal
    RESERVE AUX OPERATORS :
    - KICK : supprime un user du channel (KICK #general bob)
    - INVITE : invite un user (INVITE bob #secret)
    - TOPIC : affiche ou change le sujet du channel(TOPIC #general : Bienvenue)
    - MODE : modifie les parametres du channel
        - i : seuls les invites peuvent entrer (met ou supprime ce mode)
        - t : seul les operateurs changent le TOPIC (met ou supprime ce mode)
        - k : mdp du channel (met ou supprime ce mode)
        - o : donner ou retirer les privileges d'operateur
        - l : limite du nombre d'utilisateurs (met ou supprime ce mode)




The server creates a socket, binds it to a port,
and starts listening for incoming connections.

poll() is then used to monitor all sockets,
including the server socket and connected clients.

When a new client connects,
the server accepts the connection and creates
a dedicated socket for that client.

The client must then authenticate using
PASS, NICK and USER commands.

Once authenticated,
the client can communicate through channels
and use IRC commands."

VISION MENTALE:

SOCKET() : création de la porte
    - socket(int domain, int type, int protocol)
    - domain : AF_INET pour IPv4
    - type : SOCK_STREAM pour TCP
    - protocol : 0 pour protocol par defaut
    - return : un int qui est le fd du socket, ou -1 si ERREUR

BIND() : donner une adresse à la porte
    - bind(int socketfd, const struct sockaddr *addr, socklen_t addrlen)
    - socketfd : le retour de socket()
    - *addr : structure contennant l'adresse et le port
    - addrlen : la taille du structure (sizeof(sockaddr_in))
    EX: 
        sockaddr_in serverAddr;

        serverAddr.sin_family = AF_INET; (le type d'adresse reseau = IPv4)
        serverAddr.sin_addr.s_addr = INADDR_ANY; (ecouter sur tout les IPs de la machine)
        serverAddr.sin_port = htons(6667); (le port reseau)

        bind(serverFd, (sockaddr *)&serverAddr, sizeof(serverAddr));
    - return : 0 mais -1 si ERREUR

LISTEN() : ouvrir la porte et attendre
    - listen(int socketfd, int backlog)
    - socketfd : fd du socket du serveur par socket()
    - backlog : le nombre maximum de connection en attente
    - return : 0 mais -1 si ERREUR

POLL() : surveiller la porte et tous les clients
    - poll(struct pollfd *fds, nfds_t nfds, int timeout)
    - surveille TOUT les socket simultanement
    - *fds : un tableau de pollfd structure
    - nfds : le nombre d'element dans le tableau
    - timeout : combien de temps poll attend (-1 pour attend a l'infini)
    - struct pollfd
        {
            int fd,
            short events;
            short revents;
        }
        - fd : fd du socket a surveiller
        - events : l'evenement a surveille, souvent POLLIN pour dire "dis moi si il y a des donnees a lire"
        - revents : les evenements que poll() a detecter
        - revents & POLLIN : pour verifier si POLLIN est dans les evenement detecte
    EX:
        std::vector<pollfd> pollFds;

        pollfd serverPoll;
        serverPoll.fd = serverFd;
        serverPoll.events = POLLIN;
        serverPoll.revents = 0;

        pollFds.push_back(serverPoll);

        poll(pollFds.data(), pollFds.size(), -1);
    - return : nombre de sockets ayant un evenement, -1 ERREUR

ACCEPT() : faire entrer un client
    - accept(int socketfd, struct sockaddr *addr, socklen_t *addrlen)
    - cree une nouvelle socket pour le client
    - socketfd : socket du serveur
    - *addr : structure pour garder les informations sur l'addresse du client
    - *addrlen : la taille de la structure
    EX:
        sockaddr_in clientAddr;
        socklen_t len = sizeof(clientAddr);

        int clientFd = accept(serverFd, (sockaddr *)&clientAddr, &len);
    - return : nouvelle socket client ou -1 si ERREUR (PEUX ECHOUER TEMPORAIREMENT)

RECV() : écouter ce qu’il dit
    - recv(int socketfd, void *buf, size_t len, int flags)
    - recevoir des donnees sur un socket
    - socketfd : socket du client
    - buf : ou on stocke les donnees recues
    - len : le nombre maximum de bytes qu'on listen
    - flags : souvent 0
    EX:
        char buffer[1024];

        ssize_t bytes = recv(clientFd, buffer, sizeof(buffer), 0);
    - return : bytes des donnees recues ou 0 SI CLIENT DECONNECTE, -1 si ERREUR

SEND() : lui répondre
    - send(int socketfd, const void *buf, size_t len, int flags)
    - envoie des donnees sur un socket
    - socketfd : le socket du client
    - *buf : les donnees a lire
    - len : nombre de bytes
    - souvent 0
    EX:
        send(clientFd, message.c_str(), message.size(), 0);
    - return : nombre d'octets envoyes, -1 si ERREUR

CLOSE() : fermer la/une porte
    - close (int socketfd)

FCNTL() : utiliser pour que les sockets sont NON_BLOQUANT
    - fcntl (int fd, int cmd, ...)
    - EX : fcntl(serverFd, F_SETFL, O_NONBLOCK);
    - POURQUOI? : parceque quand recv() le serveur attends des donnees sans le mode NON_BLOQUANT
    - placer juste apres socket() ou accept()





ETAPES :

0) Vérification des arguments

    - Vérifier:
        ./ircserv <port> <password>

    - Vérifier que:
        - le port est valide
        - le mot de passe n’est pas vide


1) Préparation du serveur

    - socket()
        → créer la socket serveur

    - fcntl(..., O_NONBLOCK)
        → rendre la socket non-bloquante

    - bind()
        → associer la socket au port/IP

    - listen()
        → mettre le serveur en attente de connexions

    - créer le pollfd du serveur

    - ajouter le pollfd serveur dans:
        std::vector<pollfd> _pollFds


2) Boucle principale du serveur

    while (running)

    - poll()
        → surveiller tous les fd


3) Parcourir tous les pollfd

    Vérifier quels fd ont un événement réseau.


4) Si événement sur la socket serveur

    → un nouveau client veut se connecter

    - accept()
        → créer une nouvelle socket client

    - fcntl(..., O_NONBLOCK)
        → rendre la socket client non-bloquante

    - créer un User

    - ajouter le User dans:
        std::map<int, User*> _users

    - créer un pollfd client

    - ajouter le pollfd client dans _pollFds


5) Si événement sur une socket client

    → le client envoie des données

    - recv()

    Cas possibles:

        a) recv == 0
            → client déconnecté

            - close(fd)
            - supprimer du poll
            - supprimer du _users

        b) recv < 0
            → erreur réseau

        c) recv > 0
            → données reçues


6) Gestion du buffer utilisateur

    - ajouter les données reçues dans:
        user->_buffer

    Exemple:
        user->_buffer += recvData


7) Reconstruction des commandes IRC

    Chercher:
        \\r\\n

    Car une commande IRC complète se termine par:
        \\r\\n


8) Extraction des commandes

    Tant qu’il existe un \\r\\n:

        - extraire UNE commande complète

        - supprimer cette commande du buffer

        - parser la commande

        - exécuter la commande


9) Parsing IRC

    Transformer:

        PRIVMSG #42 :hello

    en:

        commande:
            PRIVMSG

        arguments:
            #42
            hello


10) Exécution des commandes

    Exemples:

        PASS
        NICK
        USER
        JOIN
        PRIVMSG
        KICK
        TOPIC
        MODE




COMMAND :
PRIVMSG <target>(channel or user) :message
KICK #channel user :reason (can kick even no user)
INVITE user #channel
TOPIC #channel :new topic
MODE #channel +i (after kick, can always join, invite ticket still valid)
MODE #channel +t
MODE #channel +k password CAN ENTER EVEN NO PASS
MODE #channel +o user CANT GIVE THE OP
MODE #channel +l 10 CAN ALWAYS ENTER
+i → invite only
+t → topic protégé
+k → mot de passe
+o → donner op
+l → limite utilisateur
